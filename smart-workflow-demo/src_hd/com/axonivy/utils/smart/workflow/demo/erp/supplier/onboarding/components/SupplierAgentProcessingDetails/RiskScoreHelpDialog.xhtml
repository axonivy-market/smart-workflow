Three components are each scored 0–100, then averaged into the aggregate. All three are computed deterministically by code from extracted documents and rule findings — no AI estimation involved.

Financial Stability Code
Starts at 100 — computed deterministically from financial rule findings
FAILURE finding → deducts that rule's full configured risk score
WARNING finding → deducts half of that rule's configured risk score
Each document type's worst finding counts once; clamped to 0–100
Policy Compliance Code
Starts at 100 — computed deterministically from policy findings
FAILURE finding → deducts that rule's full configured risk score
WARNING finding → deducts half of that rule's configured risk score
Clamped to 0–100; overrides the AI's own estimate
Certification Validity Code
Starts at 100
−30 if Commercial Register not submitted
−30 if required Certification missing (volume > €50k or manufacturing)
−15 if Self-Declaration not submitted
−15 if Annual Report not submitted
−40 per expired certificate (valid-until date is in the past)
−10 per certificate expiring within 6 months
−20 per certification FAILURE finding not already counted above
aggregate = floor((Financial Stability + Policy Compliance + Cert Validity) ÷ 3) · all components clamped 0–100
≥ 80 — GREEN → Approval
45–79 — YELLOW → Clarification
< 45 — RED → Decline
